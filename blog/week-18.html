Week 18: Android JNI Plumbing with CMake

This week we worked on the JNI (Java Native Interface) bridge. To call our optimized C++ libraries (libciqual_api and libuser_profile) from Kotlin, we need to compile them using the Android NDK (Native Development Kit).

The architectural layout of VITA's native bridge, connecting high-level Android Kotlin controllers to low-level C++ engines via the NDK JNI boundary.

We configured CMake inside our Android gradle structure. We wrote JNI wrapper files in C++ (ciqual_jni.cpp) that map Kotlin method calls to our C ABI. We handled standard conversions between Kotlin strings and C++ const char* arrays safely, making sure to avoid memory leaks.

We compiled the shared libraries (.so files) for the target Android architectures (specifically arm64-v8a for modern physical phones and x86_64 for emulator testing). The compilation succeeded, and we successfully loaded the native libraries in Kotlin using System.loadLibrary().
