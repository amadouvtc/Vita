Week 22: Training the Intent Classifier for Phone Deployment

In our initial architecture plan, we intended to use a large language model (LLM) as the core orchestrator. The LLM was supposed to read the raw input, classify the user's intent, and then route execution to the database nodes. This week, we realized that was a massive waste of mobile GPU compute and battery.

Instead, we built a multi-tier routing architecture. We trained a lightweight **BERT classifier** (and its French equivalent, CamemBERT) and exported it to ONNX format (only 12MB). The classifier runs first, identifying the intent in under 50ms with virtually zero battery drain.

Once the intent is determined, the heavy LLM's only job is to act as a structured parser to translate text into formatted JSON (entity extraction). This decouples the orchestration task, significantly speeds up response times, and keeps the user interface incredibly responsive.
