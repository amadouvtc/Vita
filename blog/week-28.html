Week 28: Optimizing Meal Plans and Launching the Diagnostics Web GUI

This week was all about integration and diagnostics. We implemented our core libmealplan optimizer in C++, designed fully deterministic scaling within slot windows, and connected our local voice pipelines into a sleek control panel.

To test these components end-to-end, we built a Web Diagnostic GUI using Streamlit that runs locally on port 8501. The GUI communicates directly with the C++ voice server's REST API on port 9206. It allows us to view and trigger Whisper engine toggles (local vs online), browse recipe lists with real-time fridge-stock availability badges (🟢/🟡/🔴), and debug multi-intent queries.

The real-time diagnostics workflow: a local Streamlit web application communicates directly with the C++ voice server over localhost port 9206, executing pipeline tests and returning ordered intent segments.

Furthermore, we completed the training of our lightweight BERT Sequence Classifier. The classifier fine-tunes a DistilBERT model on dataset.json containing custom health commands. The trained model is exported to ONNX format (model.onnx) and loaded via ONNX Runtime inside the voice server, allowing VITA to split compound utterances (e.g. "check protein in chicken and log 150g rice") into distinct intents in under 50ms.
